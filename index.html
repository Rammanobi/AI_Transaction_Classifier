Intelligent Financial Data Pipeline

An asynchronous, highly-scalable architecture that ingests, cleans, validates, and classifies transactions using OpenAI's LLMs.

🏗️ System Architecture

The system is built on a robust event-driven microservice architecture designed for maximum throughput and separation of concerns.

  • API Layer: FastAPI handles highly concurrent incoming HTTP connections instantly.
  • Message Broker: Redis intercepts workloads, ensuring the API is never blocked by heavy processing.
  • Worker Layer: Celery consumers process the heavy CSV parsing, data validation, and external LLM network calls asynchronously.
  • Database: PostgreSQL enforces strict relational integrity for the final audited financial data.

🔄 The 6-Stage Data Flow

1. Ingestion

Users upload a CSV. FastAPI saves it to a shared volume, logs a PENDING job to Postgres, and queues a message in Redis.

2. Parsing & Cleaning

Celery safely reads the file using Pandas, managing None types cleanly. Dates, strings, and currencies are stripped and normalized.

3. Strict Validation

Required fields and duplicate IDs are strictly checked. Failures are instantly shunted to an error log so clean data can proceed.

4. LLM Classification

Valid rows missing categories are batched to OpenAI. A strict JSON schema prompt guarantees exactly one category prediction per row.

5. Anomaly Detection

Transactions are flagged if they exceed 3x the account's median spend or if domestic merchants unexpectedly charge in USD.

6. Aggregation

Total spends are calculated securely—strictly omitting FAILED or PENDING transactions. Data is finally committed to PostgreSQL.

⚡ Setup & Quick Start

1. Environment

Create a .env file in the project root with your OpenAI API Key:

OPENAI_API_KEY=your_actual_api_key_here

2. Booting up Docker

Bring up Postgres, Redis, FastAPI, and Celery instantly:

docker compose up -d --build

API Testing via cURL

Upload CSV

curl -X POST "http://localhost:8000/jobs/upload" \
  -H "accept: application/json" \
  -H "Content-Type: multipart/form-data" \
  -F "file=@transactions.csv"

Fetch Results

curl -X GET "http://localhost:8000/jobs/{job_id}/results" \
  -H "accept: application/json"
